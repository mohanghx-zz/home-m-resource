# home-m-resource

## 项目简介

`home-m-resource` 是一个用于**归档和发布多个独立 HTML 页面**的仓库。  
本仓库不针对任何特定域名或网站，仅作为静态页面资源的集合地，方便统一管理、版本回溯与分发。

适合存放各类独立功能页面、单页应用、演示页面、工具页面或轻量级文档界面等。

## 目录结构

项目根目录下按类型或用途组织页面，建议遵循以下结构（可根据实际情况调整）：

```
home-m-resource/
├── pages/          # 存放所有 HTML 页面
│   ├── page-a/
│   │   └── index.html
│   └── page-b/
│       └── index.html
├── assets/         # 公共资源（CSS、JS、图片等）
├── docs/           # 文档或说明文件
└── README.md       # 本文件
```

> 每个独立页面建议放在单独的子目录中，便于维护和引用资源。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可直接在浏览器中打开运行。
- 页面之间互不依赖，可单独更新或删除。
- 页面中引用的资源（如样式、脚本、字体等）建议使用相对路径，或统一放在 `assets/` 目录下。
- 如需引用外部资源（如 CDN），请确保其可用性与稳定性。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/home-m-resource.git
   ```
2. 直接在浏览器中打开 `pages/` 下任意页面的 `index.html` 即可预览。
3. 如需部署到静态托管服务（如 GitHub Pages、Netlify 等），可将仓库根目录或 `pages/` 目录作为站点根目录。
